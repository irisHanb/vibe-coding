## prompt 작성시

- prompt 가 명확해야 한다. ( 매우, 아주, 섬세하고 자세하게, 그렇지 않을 경우 꼬
  인다... )
  - 공통 룰 작성
  - 작업마다의 고유 룰 작성
- 설계는 개발자가 한다.
  - 설계는 개발자가 하고 모듈식으로 적절한 단위로의 작업을 분할, 진행한다.
- prompt 실행 후 반드시!!! 룰 재검토를 요청하여 구현시 오류들을 재확인 수정할 수
  있도록 한다.
